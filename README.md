# KET 单词检验

一个纯前端、可离线打开的单词检验小工具。  
用于把词库中的单词按学习状态分类为：

- 待处理
- 已完成
- 继续学习

支持本地持久化、导入导出、进度重置，以及当前单词查询（音标 + 英音/美音发音）。

## 功能列表

- 单词检验流程：已完成 / 继续学习 / 跳过 / 撤销上一步
- 进度统计：总词数、待处理、已完成、继续学习、完成率
- 本地缓存：自动保存进度到浏览器 `localStorage`
- 导出：将当前进度导出为 `md` 文件
- 导入：通过 `md` 文件覆盖当前进度
- 重置：清除本地缓存并恢复初始状态
- 查询当前词：弹窗显示释义（词典兜底）和英音/美音播放

## 项目文件

- `index.html`：页面结构
- `style.css`：页面样式
- `app.js`：核心逻辑
- `ket-1975.md`：词库源文件

## 快速开始

### 本地起服务访问（推荐）

在项目目录执行：

```bash
python3 -m http.server 8000
```

然后访问：`http://localhost:8000`

说明：
- 当前版本直接读取 `ket-1975.md`
- 建议使用本地 HTTP 服务，不建议 `file://` 直接打开

## 词库格式

导入/初始化识别以下 Markdown 结构：

```md
## 未验证
apple
book

## 会背
cat

## 不会背
language
```

注意：
- 标题必须是：`## 未验证` / `## 会背` / `## 不会背`
- 每行一个单词

## 查询功能说明

- 点击“查询当前词”会弹出词汇详情
- 显示内容：
  - 单词
  - 释义（当前为词典释义兜底，不强依赖在线翻译接口）
  - 英音音标 / 美音音标
- 发音：
  - 支持“播放英音”“播放美音”
  - 若词典只返回一种发音，会自动禁用另一按钮

## 数据存储与重置

- 存储键：`ket1975.progress.v1`
- 每次操作会自动保存
- 点击“重置进度”会：
  - 清除本地缓存
  - 恢复所有单词为“待处理”
  - 清空操作历史

## 已知限制

- 纯前端静态页面，未接后端
- 在线翻译接口在部分网络环境下不可达，因此当前默认不依赖在线翻译
- 发音与音标依赖外部词典接口可用性

## 后续可选优化

- 接入自建后端翻译代理（提升国内网络稳定性）
- 查询结果缓存（减少重复请求）
- 增加例句、词性、常见搭配展示
